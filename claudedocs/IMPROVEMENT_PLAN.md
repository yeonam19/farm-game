# 힐링 농장 개선 계획서

## 개요

현재 코드베이스 분석 결과를 바탕으로 11개 개선 항목을 우선순위별로 정리한 계획서입니다.
각 항목은 독립적으로 작업 가능하며, 순차적으로 진행합니다.

---

## Phase 1: 핵심 개선 (높은 우선순위)

### #1 사운드/오디오 시스템 추가
- **현상**: BGM, 효과음이 전혀 없음
- **목표**: 힐링 분위기에 맞는 오디오 시스템 구축
- **작업 내용**:
  - `core/src/.../system/AudioSystem.kt` 생성
  - LibGDX 내장 `Music`/`Sound` API 활용 (별도 의존성 불필요)
  - BGM: 계절별 배경음악 (Spring/Summer/Autumn/Winter)
  - SFX: 밭갈기, 물주기, 수확, 낚시, 구매/판매, UI 클릭
  - 볼륨 조절 기능 (BGM/SFX 분리)
  - `assets/audio/bgm/`, `assets/audio/sfx/` 디렉토리 구성
  - TimeSystem 콜백과 연동하여 계절 변경 시 BGM 전환
- **관련 파일**:
  - 신규: `AudioSystem.kt`
  - 수정: `GameWorld.kt` (시스템 초기화), `GameScreen.kt` (SFX 트리거), `FarmGame.kt` (리소스 해제)
- **예상 규모**: 파일 1개 신규, 3개 수정
- **상태**: ⏳ 대기

### #2 대형 파일 분리 (리팩토링)
- **현상**: `GameScreen.kt` 1,908줄, `GameWorld.kt` 1,132줄
- **목표**: 단일 책임 원칙에 따라 기능별 분리
- **작업 내용**:
  - **GameScreen.kt 분리**:
    - `renderer/WorldRenderer.kt` — 타일, 작물, 나무/바위 렌더링
    - `renderer/CharacterRenderer.kt` — 플레이어, 농부, 목동 렌더링 (중복 제거 포함)
    - `renderer/WeatherRenderer.kt` — 비/눈/폭풍 파티클 렌더링
    - `renderer/UIRenderer.kt` — 인게임 오버레이 (도구 표시, 상호작용 안내)
    - `input/InputHandler.kt` — 키보드/마우스 입력 처리
    - `GameScreen.kt` — 렌더러 조합 + 게임 루프만 유지
  - **GameWorld.kt 분리**:
    - `world/MapGenerator.kt` — 맵 생성, 나무/바위 배치
    - `world/FarmManager.kt` — 농사 관련 로직 (밭갈기, 물주기, 수확)
    - `world/WorkerManager.kt` — 고용 일꾼 AI, 작업 할당
    - `world/FishingManager.kt` — 낚시/자동낚시 로직
    - `GameWorld.kt` — 매니저 조합 + 상태 관리만 유지
- **관련 파일**:
  - 신규: 9개 파일
  - 수정: `GameScreen.kt`, `GameWorld.kt` (대폭 축소)
- **예상 규모**: 대형 리팩토링
- **상태**: ⏳ 대기

### #3 게임 밸런스 조정
- **현상**: 경제/경험치/자동화 밸런스 불균형
- **목표**: 자연스러운 진행 곡선 설계
- **작업 내용**:
  - **경제 밸런스**:
    - 시작 골드: 500 → 1000
    - 첫 도구 업그레이드: 500 → 800
    - 스프링클러 Lv1: 2000 → 1500
    - 농부 고용비 5000, 목동 고용비 4000 → 각각 4000, 3500으로 조정
  - **경험치 밸런스**:
    - 작물별 차등 경험치 (성장시간 비례): 옥수수 15 EXP, 시금치 8 EXP 등
    - 레벨업 공식: `100 + level * 80` → `100 + level * 60` (초중반 완화)
  - **낚시 밸런스**:
    - 수동 낚시 희귀 확률: 5% → 8% (자동낚시 Lv3과 동일)
    - 자동낚시 Lv3 비용: 25000 → 20000
  - **스프링클러 조정**:
    - 작물 무수분 생존: 3일 → 2일
    - 스프링클러 Lv1은 격일 작동으로 변경
- **관련 파일**:
  - 수정: `GameData.kt`, `GameWorld.kt`, `MissionSystem.kt`, `Crop.kt`
- **예상 규모**: 4개 파일 수정 (수치 변경 중심)
- **상태**: ⏳ 대기

### #4 매직 넘버 상수화
- **현상**: 20개 이상의 하드코딩 값이 여러 파일에 산재
- **목표**: `GameConfig` 객체로 중앙 관리
- **작업 내용**:
  - `data/GameConfig.kt` 생성 — 모든 게임 상수 집중 관리
  - 카테고리별 정리:
    - `MapConfig`: 맵 크기(50x50), 타일 크기(48f), 농장 경계
    - `TimeConfig`: 하루 길이(720f), 계절 일수(28), 주야 범위
    - `WeatherConfig`: 비/눈 파티클 수, 속도, 폭풍 강도
    - `FishingConfig`: 낚시 확률, 자동낚시 간격
    - `WorkerConfig`: 일꾼 행동 범위, 작업 간격
    - `RenderConfig`: ShapeRenderer 용량, 카메라 기본값
  - 기존 파일에서 매직 넘버를 `GameConfig.XXX`로 교체
- **관련 파일**:
  - 신규: `GameConfig.kt`
  - 수정: `GameScreen.kt`, `GameWorld.kt`, `TimeSystem.kt`, `WeatherSystem.kt`
- **예상 규모**: 파일 1개 신규, 4~5개 수정
- **상태**: ⏳ 대기

## Phase 2: 품질 개선 (중간 우선순위)

### #5 캐릭터 렌더링 중복 제거
- **현상**: 플레이어/농부/목동 렌더링이 거의 동일한 코드 3벌
- **목표**: 공통 렌더링 메서드 추출
- **작업 내용**:
  - `renderer/CharacterRenderer.kt`에 공통 `renderCharacter()` 메서드
  - 파라미터: 위치, 방향, 색상 팔레트, 애니메이션 상태, 도구 종류
  - 플레이어/농부/목동은 팔레트와 도구만 다르게 전달
  - 기존 150줄+ 중복 → 50줄 이내로 축소
- **선행 조건**: #2 (리팩토링) 완료 후 진행
- **예상 규모**: 1개 파일 수정
- **상태**: ⏳ 대기

### #6 UX 피드백 강화
- **현상**: 작물 성장률, 동물 행복도, 일꾼 상태 등 미표시
- **목표**: 핵심 정보를 플레이어에게 시각적으로 전달
- **작업 내용**:
  - **작물 성장 게이지**: 작물 위에 초록색 진행 바 표시
  - **동물 상태 아이콘**: 배고픔(❗), 생산 가능(✨) 아이콘
  - **일꾼 상태 표시**: 일꾼 머리 위 현재 작업 아이콘 (물뿌리개/낫/먹이)
  - **도구 레벨 표시**: 도구 아이콘 옆 별(★) 개수로 등급 표시
  - **날씨 예보**: HUD에 내일 날씨 아이콘 추가
- **관련 파일**:
  - 수정: `GameScreen.kt` (또는 리팩토링 후 각 Renderer), `GameHUD.kt`
- **예상 규모**: 2~3개 파일 수정
- **상태**: ⏳ 대기

### #7 에러 처리 개선
- **현상**: 예외를 조용히 무시하거나 printStackTrace만 호출
- **목표**: 안정적 에러 처리 + 로깅 시스템
- **작업 내용**:
  - `system/Logger.kt` 생성 — 파일 기반 로깅 (기존 crash_log.txt 확장)
  - `SaveSystem.kt`: 저장/로드 실패 시 사용자 알림 + 백업 복구 시도
  - `GameHUD.kt`: 미션 아이템 파싱 에러 시 기본값 처리 + 로그 기록
  - `GameWorld.kt`: NPC 상호작용 null-safety 강화
- **관련 파일**:
  - 신규: `Logger.kt`
  - 수정: `SaveSystem.kt`, `GameHUD.kt`, `GameWorld.kt`
- **예상 규모**: 파일 1개 신규, 3개 수정
- **상태**: ⏳ 대기

### #8 NPC 시스템 확장
- **현상**: NPC 위치 하드코딩, 대화 진행/호감도 없음
- **목표**: 의미 있는 NPC 상호작용
- **작업 내용**:
  - 대화 진행 시스템: 미션 진행도에 따라 대사 변경
  - 호감도 시스템: 상호작용 횟수에 따른 보상 해금
  - NPC 위치를 MapGenerator로 이동 (하드코딩 제거)
  - 상인 NPC: 호감도 상승 시 할인 제공
- **관련 파일**:
  - 수정: `NPC.kt`, `GameData.kt`, `GameWorld.kt`
- **예상 규모**: 3개 파일 수정
- **상태**: ⏳ 대기

## Phase 3: 폴리시 (개선하면 좋은 것)

### #9 스프라이트/텍스처 도입
- **현상**: ShapeRenderer로 모든 그래픽을 직접 그리는 중
- **목표**: 스프라이트 기반 렌더링으로 비주얼 품질 향상
- **작업 내용**:
  - 픽셀아트 스프라이트 시트 제작 (캐릭터, 작물, 타일, UI)
  - TextureAtlas 기반 렌더링으로 전환
  - SpriteBatch 활용으로 성능 개선
  - 애니메이션 프레임 지원
- **예상 규모**: 대형 작업 (아트 에셋 필요)
- **상태**: ⏳ 대기

### #10 설정 파일 외부화
- **현상**: 모든 밸런스 파라미터가 코드에 하드코딩
- **목표**: JSON 기반 데이터 관리로 밸런스 조정 용이화
- **작업 내용**:
  - `assets/data/crops.json` — 작물 정의
  - `assets/data/animals.json` — 동물 정의
  - `assets/data/upgrades.json` — 업그레이드 정의
  - `assets/data/missions.json` — 미션 체인 정의
  - `DataLoader.kt` 생성 — JSON 파싱 및 검증
- **예상 규모**: 파일 5개 신규, 다수 수정
- **상태**: ⏳ 대기

### #11 추가 콘텐츠
- **현상**: 기본 농장 루프만 존재
- **목표**: 게임 깊이 확장
- **후보 기능**:
  - 요리/가공 시스템 (작물 → 가공품, 높은 판매가)
  - 날씨 예보 시스템 (TV/라디오 NPC)
  - 일꾼 관리 UI (배치, 상태 확인, 해고)
  - 난이도 설정 (Easy/Normal/Hard)
- **예상 규모**: 항목별 개별 판단
- **상태**: ⏳ 대기
